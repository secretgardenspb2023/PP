# Yandex Object Storage (S3) для PoiskPlant — инструкция

Нужно создать хранилище для фотографий растений и выдать разработчику **3 значения**:
**Access Key ID**, **Secret Access Key**, **имя бакета**. Делается в аккаунте заказчика
(привязка карты и оплата — на стороне заказчика; объём расходов копеечный).

> Если разработчик приглашён в вашу организацию — создавать платёжный аккаунт и ресурсы
> должны **вы как владелец** (у приглашённого нет прав, будет «Permission denied»).

---

## Шаг 1. Платёжный аккаунт
1. Открыть **console.yandex.cloud** → блок **Billing** → **«Создать платёжный аккаунт»**.
2. Тип — **Физическое лицо** (или юрлицо), заполнить данные, привязать карту
   (спишется ~1 ₽ для проверки и вернётся). Обычно даётся пробный грант.

## Шаг 2. Облако и каталог
После биллинга в консоли появится **облако** (Cloud) и внутри **каталог** (folder) —
создаются автоматически или кнопкой «Создать».

## Шаг 3. Бакет (хранилище)
1. В консоли: **Object Storage** → **Создать бакет**.
2. Имя: например **`poiskplant-media`** (только латиница/цифры/дефис, уникальное).
3. Доступ: **Приватный** (не публичный).
4. Создать.

## Шаг 4. Сервисный аккаунт + ключ (это и есть доступы)
1. В консоли: **Сервисные аккаунты** → **Создать сервисный аккаунт** (имя, напр. `poiskplant-app`).
2. Назначить роль **`storage.editor`** (на каталог или бакет).
3. Открыть этот сервисный аккаунт → **«Создать новый ключ»** → **«Статический ключ доступа»**.
4. Появятся **Access Key ID** и **Secret access key**.
   ⚠️ **Secret показывается один раз** — скопируйте сразу.

## Шаг 5. Что прислать разработчику
- **Access Key ID** (вида `YCAJE…`)
- **Secret access key** (длинная строка)
- **Имя бакета** (напр. `poiskplant-media`)

Отправить личным сообщением (в Telegram), **не публиковать** публично.

---

*Endpoint и регион стандартные (`storage.yandexcloud.net`, `ru-central1`) — отдельно сообщать не нужно.*
