# Инструкции по регистрации внешних сервисов

## Проект PoiskPlant

Приложение к ТЗ v2.2. Все сервисы регистрируются на аккаунт заказчика. После регистрации заказчик передаёт исполнителю только необходимые ключи и токены (перечислены в каждом разделе). При возникновении вопросов в процессе регистрации — пишите, помогу.

**Что регистрируем:**

- Google Cloud Console — OAuth-вход через Google
- VK Developers — OAuth-вход через ВКонтакте
- Telegram BotFather — OAuth-вход через Telegram
- Yandex SmartCaptcha — защита форм от ботов
- Yandex Mail для домена — отправка писем (регистрация, восстановление пароля) — бесплатно
- Яндекс.Метрика — веб-аналитика
- Sentry — мониторинг ошибок

---

## 1. Google Cloud Console — OAuth для входа через Google

**Время регистрации: ~20 минут.**

**Шаги:**

1. Перейдите на https://console.cloud.google.com/ и войдите под Google-аккаунтом, на который будете регистрировать проект.
2. В верхней панели нажмите «Select a project» → «New Project». Имя проекта: PoiskPlant. Нажмите «Create».
3. После создания проекта откройте боковое меню (≡) → «APIs & Services» → «OAuth consent screen».
4. Выберите тип «External» (для публичных пользователей) → «Create».
5. Заполните: App name — PoiskPlant; User support email — ваш email; Developer contact email — ваш email. Остальное можно пропустить, нажать «Save and Continue».
6. На шаге «Scopes» нажмите «Add or Remove Scopes», выберите `userinfo.email` и `userinfo.profile`. Сохранить.
7. На шаге «Test users» добавьте свой email (для тестирования до публикации). Сохранить.
8. Откройте «APIs & Services» → «Credentials» → «Create Credentials» → «OAuth client ID».
9. Application type — «Web application». Name — PoiskPlant Web.
10. Authorized JavaScript origins — пока добавьте: `http://localhost:3000` и `https://staging.вашдомен.ру` (точный URL пришлю позже).
11. Authorized redirect URIs — пока добавьте: `http://localhost:8000/api/v1/auth/google/callback/` и `https://staging.вашдомен.ру/api/v1/auth/google/callback/` (точный URI пришлю позже).
12. Нажмите «Create». Появится окно с двумя строками: Client ID и Client Secret.

**Что передать исполнителю:**

- Client ID (длинная строка, заканчивается на `.apps.googleusercontent.com`)
- Client Secret (короткая строка)

*Передать в Telegram или зашифрованным сообщением. Не публиковать публично.*

## 2. VK Developers — OAuth для входа через ВКонтакте

**Время регистрации: 10 минут + до 1–3 дней модерация со стороны ВКонтакте.**

> **ВНИМАНИЕ: VK может тормозить с модерацией. Зарегистрировать приложение в первый день работ.**

**Шаги:**

1. Перейдите на https://id.vk.com/about/business/go и войдите под VK-аккаунтом.
2. Если работаете как физлицо — выберите соответствующий вариант. Если есть юрлицо — можно зарегистрировать на него.
3. Нажмите «Создать приложение». Название — PoiskPlant. Платформа — «Веб-сайт».
4. Адрес сайта — пока ваш текущий домен (poiskplant.ru), позже добавим staging.
5. Базовый домен — `poiskplant.ru`.
6. После создания откройте раздел «Настройки» приложения.
7. В «Доверенные redirect URI» добавьте: `https://poiskplant.ru/api/v1/auth/vk/callback/`, `https://staging.вашдомен.ру/api/v1/auth/vk/callback/`.
8. Найдите раздел «Ключи доступа» → «Защищённый ключ» (Service Token).

**Что передать исполнителю:**

- ID приложения (App ID, число)
- Защищённый ключ (Service Token)
- Защищённый ключ (Secure Key, если есть отдельно)

## 3. Telegram BotFather — OAuth для входа через Telegram

**Время регистрации: 5 минут.**

**Шаги:**

1. Откройте Telegram, найдите в поиске @BotFather и начните диалог.
2. Отправьте команду `/newbot`.
3. BotFather спросит имя бота — введите: `PoiskPlant Login`.
4. Затем спросит username бота — введите: `poiskplant_login_bot` (должен заканчиваться на `_bot` и быть уникальным; если занят, придумайте вариацию, например `poiskplant_auth_bot`).
5. BotFather пришлёт сообщение с токеном вида `1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ`. Это ваш Bot Token, сохраните его.
6. Отправьте команду `/setdomain`.
7. BotFather попросит выбрать бота — выберите только что созданного.
8. Введите домен — пока `poiskplant.ru`. Позже добавим staging.
9. Готово. Бот настроен для Telegram Login Widget.

**Что передать исполнителю:**

- Bot Token (длинная строка с двоеточием)
- Username бота (например, `poiskplant_login_bot`)

## 4. Yandex SmartCaptcha — защита от ботов

**Время регистрации: 10 минут.**

**Шаги:**

1. Перейдите на https://console.cloud.yandex.ru/ и войдите под Yandex-аккаунтом.
2. Если ещё не зарегистрированы в Yandex Cloud — пройдите регистрацию (потребуется привязать платёжный аккаунт, но SmartCaptcha в бесплатных лимитах).
3. Создайте «Платёжный аккаунт» (просто заполнить данные, оплата только при превышении бесплатного лимита).
4. В каталоге найдите сервис «SmartCaptcha».
5. Нажмите «Создать капчу». Название — PoiskPlant.
6. Тип — «Невидимая капча» (рекомендуется, не мешает пользователям) или «С челленджем» (показывает картинки) — на ваш выбор.
7. Список разрешённых доменов: `poiskplant.ru`, `staging.вашдомен.ру`, `localhost`.
8. Нажмите «Создать».
9. После создания на странице капчи будут два значения: Client key (открытый, для фронта) и Server key (секретный, для бэка).

**Что передать исполнителю:**

- Client key (открытый ключ для встраивания на фронт)
- Server key (секретный ключ для проверки на бэке)

## 5. Yandex Mail для домена — email для писем регистрации и уведомлений

**Время регистрации: 30 минут + до 1 суток на распространение DNS.**

*Это бесплатный сервис от Яндекса для отправки писем с собственного домена. Не требует оплаты, лимит 500 писем/сутки на ящик хватит для регистраций и восстановления паролей на годы вперёд. Доставляемость отличная.*

**Шаги:**

1. Перейдите на https://360.yandex.ru/business/mail/ и нажмите «Подключить почту».
2. Введите домен `poiskplant.ru`. Войдите под Yandex-аккаунтом (можно создать новый специально для проекта, например `poiskplant.admin@yandex.ru`).
3. Яндекс попросит подтвердить владение доменом — покажет DNS-запись TXT с уникальным значением. Добавьте её в DNS-настройки вашего домена (там, где управляете записями).
4. После подтверждения (5–15 минут) Яндекс покажет MX-записи, которые нужно добавить в DNS. Также рекомендуется добавить SPF, DKIM, DMARC записи — они будут указаны в инструкции Яндекса.
5. Подождите до 1–24 часов, пока DNS-записи распространятся (можно проверить через https://mxtoolbox.com).
6. В кабинете Yandex Mail создайте ящик `noreply@poiskplant.ru` — это будет ящик-отправитель писем.
7. В настройках безопасности этого ящика создайте «пароль приложения» для SMTP (обычный пароль для веб-входа не подойдёт). Сохраните его.

**Что передать исполнителю:**

- Email отправителя: `noreply@poiskplant.ru`
- Пароль приложения (для SMTP)
- SMTP-сервер: `smtp.yandex.ru`, порт 465 (SSL) — это стандартные значения, на всякий случай

## 6. Яндекс.Метрика — веб-аналитика

**Время регистрации: 10 минут.**

*Бесплатный сервис веб-аналитики от Яндекса. Будет показывать количество посетителей, источники трафика, поведение на сайте.*

**Шаги:**

1. Перейдите на https://metrika.yandex.ru и войдите под Yandex-аккаунтом (тем же, что для Yandex Mail, удобнее).
2. Нажмите «Добавить счётчик».
3. Имя счётчика — PoiskPlant. Адрес сайта — `https://poiskplant.ru`. Часовой пояс — Москва (или ваш).
4. Принять условия и согласие на обработку данных.
5. Включите опции: «Вебвизор», «Карты», «Аналитика форм» (полезно для понимания поведения пользователей).
6. В настройках счётчика → «Обработка данных» → включите «Принимать данные только при наличии cookie-согласия» (мы будем загружать Метрику только после согласия пользователя в баннере).
7. Также включите анонимизацию IP-адресов (опция в настройках).
8. После создания счётчика скопируйте его номер (8-значное число, например `12345678`).

**Что передать исполнителю:**

- Номер счётчика Метрики

*Код встраивания на сайт писать не нужно — разработчик встроит счётчик в код, причём загружаться он будет только после согласия пользователя в cookie-баннере.*

## 7. Sentry — мониторинг ошибок

**Время регистрации: 10 минут.**

*Sentry автоматически собирает ошибки бэкенда и фронтенда, показывает стек-трейсы, помогает быстро находить и исправлять баги. Бесплатный тир (5000 событий/мес) хватит на старте.*

**Шаги:**

1. Перейдите на https://sentry.io/signup/ и зарегистрируйтесь.
2. Выберите бесплатный план «Developer».
3. При создании первого проекта выберите платформу Django (или Python). Название проекта — `poiskplant-backend`.
4. После создания Sentry покажет DSN — строку вида `https://abcd1234@o123456.ingest.sentry.io/7890123`. Сохраните её.
5. Создайте ещё один проект для фронтенда: Next.js. Название — `poiskplant-frontend`.
6. Сохраните DSN для фронтенда.

**Что передать исполнителю:**

- DSN для backend-проекта (Django)
- DSN для frontend-проекта (Next.js)

## Общие рекомендации

- Все ключи и токены передавать в Telegram личным сообщением, не через публичные каналы.
- Сохраните копию всех ключей и логинов в надёжном месте (например, менеджер паролей: Bitwarden, KeePass).
- Регистрировать все сервисы лучше на один email — будет проще управлять.
- Если регистрация на каком-то сервисе вызывает затруднения — напишите, разберёмся вместе. Лучше потратить 15 минут на созвон, чем неделю переписки.
- После каждой регистрации присылайте ключи отдельным сообщением с пометкой, какой это сервис.

**При появлении вопросов в процессе — пишите в Telegram, помогу пройти любой из шагов.**
