# Доступ разработчику к репозиторию на GitHub

## Зачем это нужно

Сейчас код проекта хранится в вашем репозитории на GitHub:
**`secretgardenspb2023/PP`**

Чтобы разработчик мог сохранять туда исправления и обновления (а не в отдельную
папку), ему нужен доступ **на запись** в этот репозиторий. Сейчас доступа нет —
GitHub отвечает «отказано». Из-за этого свежее исправление пока не перенесено в ваш
репозиторий (на сайте оно уже работает, но в вашем хранилище кода его ещё нет).

Нужно один раз добавить разработчика в «соавторы» репозитория. Это безопасно: вы
остаётесь владельцем, доступ в любой момент можно отозвать.

**Кого добавить (логин разработчика на GitHub):** `Cergei-ui`

---

## Как добавить (5 шагов, ~2 минуты)

1. Зайдите на GitHub под своим аккаунтом и откройте репозиторий:
   **https://github.com/secretgardenspb2023/PP**

2. Вверху страницы репозитория нажмите вкладку **Settings** (⚙️ «Настройки»).
   *(Если вкладки Settings нет — значит вы зашли не под аккаунтом-владельцем.)*

3. В левом меню выберите **Collaborators** (или **Collaborators and teams**).
   GitHub может попросить подтвердить пароль — это нормально.

4. Нажмите зелёную кнопку **Add people**, в поле введите логин:
   **`Cergei-ui`** → выберите его в подсказке → роль оставьте **Write** →
   нажмите **Add … to this repository**.

5. Готово. Разработчику придёт приглашение, он его примет — и доступ заработает.

---

## Если что-то не находится

- Вкладка **Settings** видна только владельцу/администратору репозитория.
- Раздел называется **Collaborators** или **Collaborators and teams** (зависит от
  типа аккаунта — личный или организация). В организации путь:
  **Settings → Access → Collaborators and teams → Add people**.
- Роль должна быть именно **Write** (не Read) — иначе сохранять обновления не выйдет.

## Как отозвать доступ потом (если понадобится)

Тот же раздел **Collaborators** → напротив `Cergei-ui` нажать **Remove**.

---

*После добавления напишите разработчику — он перенесёт исправление в ваш репозиторий
и настроит, чтобы будущие обновления сервера тянулись прямо оттуда.*
