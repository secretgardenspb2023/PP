# Привязка домена для PoiskPlant (staging) — инструкция по DNS

Чтобы открыть новую версию сайта по адресу **`https://staging.poiskplant.ru`**, нужно
добавить **одну DNS-запись**, которая направит этот поддомен на наш сервер.

> ⚠️ Это **не затронет** действующий сайт. Мы добавляем только поддомен `staging`.
> Сам `poiskplant.ru` и `www.poiskplant.ru` остаются как есть — старый сайт продолжает работать.

---

## Что добавить — одна запись типа A

| Поле | Значение |
|------|----------|
| **Тип записи** | `A` |
| **Имя / Хост / Subdomain** | `staging`  *(если поле просит полное имя — впишите `staging.poiskplant.ru`)* |
| **Значение / IP-адрес / Points to** | `5.101.0.240` |
| **TTL** | по умолчанию (можно поставить `600` — тогда применится быстрее) |

**Если домен подключён к Cloudflare:** при добавлении записи **отключите проксирование**
(значок «облако» должен быть **серым** — *DNS only*, не оранжевым). Иначе не выпустится
SSL-сертификат.

---

## Где это сделать

В панели управления DNS домена **poiskplant.ru** — обычно это один из вариантов:

1. **Личный кабинет регистратора**, где куплен домен → раздел «DNS», «DNS-записи»,
   «Управление зоной» или «Zone editor».
2. **Cloudflare** (если домен подключён к нему) → вкладка **DNS** → **Add record**.
3. **DNS-панель хостинга**, где размещён текущий сайт.

Порядок действий обычно такой:
1. Открыть раздел DNS-записей.
2. Нажать «Добавить запись» / «Add record».
3. Выбрать тип **A**.
4. Вписать значения из таблицы выше.
5. Сохранить.

---

## Как проверить, что заработало

- Онлайн: открыть **https://dnschecker.org**, ввести `staging.poiskplant.ru`, выбрать
  тип `A` → должно показать **`5.101.0.240`**.
- Применяется обычно за **5–30 минут** (иногда до нескольких часов).

После того как запись добавлена — сообщите разработчику, он:
1. Выпустит бесплатный SSL-сертификат (Let's Encrypt, с авто-продлением).
2. Настроит HTTPS и веб-сервер.
3. Откроет рабочий адрес **`https://staging.poiskplant.ru`** для приёмки.
